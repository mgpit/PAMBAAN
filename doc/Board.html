Bugzilla::Extension::PAMBAAN::Board

Based on Bugzilla::Object.

DESCRIPTION

SYNOPSIS

Transient Instances

    use Bugzilla::Extension::PAMBAAN::Lane;
    
    sub someMethod{
        my $adminuser = new Bugzialla::User( "admin@mydomain.tld" );
        my $lane = Bugzilla::Extension::PAMBAAN::Lane->transientnew( "First Board" );
        );
    }

Persistent Instances

    use Bugzilla::Extension::PAMBAAN::Board;
    
    sub someMethod{
        my $adminuser = new Bugzialla::User( "admin@mydomain.tld" );
        my $board = new Bugzilla::Extension::PAMBAAN::Board( "First Board");
        
        my $lane1 = Bugzilla::Extension::PAMBAAN::Lane->create( 
            {
                name                    =>  "First Board", 
                description             =>  "My first Board", 
                defaultBoard            =>  0
            }
        );
    }

CLASS METHODS

INSTANCE CREATION

As you can see from the "SYNOPSIS" and taking into account that Bugzilla::Extension::PAMBAAN::Board is a Bugzilla::Object, there are three methods for instance creation.

new and create

as inherited from "new" in Bugzilla::Object and "create" in Bugzilla::Object

transientnew

defined here

Where new can also traverse the dependency graph ({ ... with_lanes=>1 }) and gather the Bugzilla::Extension::PAMBAAN::Lanes associated with the board, create and transientnew won't do this for obvious reason.
